# Execution Harness

The harness layer provides the safety and resource constraints necessary for autonomous execution.

## `ToolHarness`
**Location:** `src/simpcode/harness/tools.py`
The harness is the only way for the assistant to interact with the file system or shell during the `TAKE ACTION` phase.
- **Scope Enforcement:** Checks every `write_file` call against the `allowed_files` list generated by the approved plan.
- **Shell Allowlist:** Restricts shell commands to a safe subset (e.g., `pytest`, `flake8`).
- **Inline Sync:** Automatically triggers `WikiEngine` hash updates after every write operation.

## `ContextBudgeter`
**Location:** `src/simpcode/harness/budgeter.py`
Manages the size of the LLM prompt.
- **Prioritization:** Mandatory files (`SIMP.md`) are always included first. Semantic and targeted context is added as budget allows.
- **Token Estimation:** Provides a rough estimate of token usage to prevent prompt overflows.

## `PermissionSystem`
**Location:** `src/simpcode/harness/permissions.py`
Handles user interactions for plan approval.
- **Rich Visualization:** Uses the `rich` library to display plans in a clear, tabular format.
- **Approval Logic:** Blocks execution until the user explicitly confirms the proposed plan.
